员工入离职 IT 流程规范 (Onboarding & Offboarding)

本文档规定了员工入职（Onboarding）与离职（Offboarding）期间，IT 部门在账号开通、设备发放、权限回收等环节的标准操作流程，确保数据安全与业务连续性。

1. 入职流程 (Onboarding)​

1.1 准备阶段 (入职前 3 天)​

  • 触发条件: 收到 HR 发送的《新员工入职通知单》（邮件或 OA 工单）。
  • 动作:
    1. 账号预创建:
      • 在 Casdoor 中创建账号，分配初始用户组（如 Group-R&D）。
      • 自动同步生成企业邮箱 (Mailcow) 和 即时通讯 (Mattermost) 账号。
      • 生成临时密码并通过短信/个人邮箱发送给员工。
    2. 设备准备:
      • 从库存提取电脑（Mac/Windows），根据岗位执行 SOE 初始化 (参考 桌面标准化规范)。
      • 在 Snipe-IT 中进行资产登记，状态更为“待发放”。
      • 在 Fleet 中注册设备，确保已安装 Agent。

1.2 入职当天​

  • 动作:
    1. 设备交付: IT 现场指导员工签收设备，签署《IT 资产使用协议》。
    2. 首次登录: 指导员工修改 Casdoor 初始密码，并绑定 MFA (多因素认证)。
    3. 权限验证: 确认员工能访问 GitLab、Nextcloud 等基础服务。
    4. 安全培训: 引导员工阅读《信息安全手册》。

2. 离职流程 (Offboarding)​

2.1 发起阶段 (离职前 1-3 天)​

  • 触发条件: 收到 HR 或审批流的离职通知。
  • 动作:
    1. 权限盘点: 检查该员工拥有的特殊权限（如云平台 Admin、服务器 Root 权限）。
    2. 数据交接: 协助部门主管进行文档 (Nextcloud) 和代码 (GitLab) 的所有权转移。

2.2 离职当天 (Last Day)​

  • 关键动作 (必须在离职时刻立即执行):
    1. 账号禁用:
      • 在 Casdoor 中将账号状态设为 Disabled (一键切断所有 SSO 应用访问)。
      • 注意: 此时保留邮箱数据，但禁止登录。
    2. VPN 吊销: 在 WireGuard 网关中移除该用户的 Peer。
    3. 设备回收:
      • 回收电脑、显示器等硬件。
      • 检查设备外观与功能。
      • 在 Snipe-IT 中更新状态为“待检测/库存”。
    4. 设备清洗:
      • 确认数据已备份后，对回收电脑进行全盘数据擦除 (Secure Erase)。
      • 重装系统，回归初始状态。

3. 异常处理​

  • 紧急离职 (红线违规): 此时应优先执行 账号禁用，再走审批流程。
  • 设备丢失: 立即通过 Fleet 执行远程锁定 (Lock) 或远程擦除 (Wipe)。

4. 审计与合规​

  • 每月 1 日生成《账号与资产一致性报告》，比对 HR 在职名单与 Casdoor 活跃账号名单。
