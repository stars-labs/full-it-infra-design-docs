推荐方案：rclone​

对于50人规模的企业，rclone 是最佳选择：

优势​

特性说明
多流传输默认4个流，带宽利用率高
带宽控制--bwlimit 精确控制
断点续传支持，检查点机制
边传边删--remove-source-files 或 --delete-during
云存储支持未来可扩展到S3/OSS等
易于配置YAML配置，无需复杂服务

rclone 配置​

# 安装
curl https://rclone.org/install.sh | sudo bash

# 配置rclone (交互式)
rclone config

# 或直接命令行使用
rclone copy /data/training remote:backup/training \
    --verbose \
    --transfers 8 \           # 并行8个传输
    --bwlimit 50M \           # 带宽限制 50MB/s
    --checksum \              # 基于checksum校验
    --delete-during \         # 传输过程中删除源文件
    --remove-source-files \   # 传输完成后删除源文件
    --log-file /var/log/rclone.log

优化后的备份脚本（rclone版）​

#!/bin/bash
# ~/scripts/backup-rclone.sh
# 训练数据备份 - rclone高速版

set -euo pipefail

# 配置
REMOTE_NAME="nasbackup"           # rclone remote名称
SOURCE_PATHS=(
    "/data/training"
    "/data/models"
    "/data/experiments"
)
BW_LIMIT="50M"                    # 带宽限制
TRANSFERS=8                       # 并行传输数
CHECKERS=16                       # 并行校验数
LOG_DIR="${HOME}/.local/share/backup/logs"
LOG_FILE="${LOG_DIR/backup_$(date +%Y%m%d).log"

# 时间窗口
START_HOUR="22"
END_HOUR="7"

log() {
    echo "[$(date '+%Y-%m-%d %H:%M:%S')] $*" | tee -a "${LOG_FILE}"
}

check_time_window() {
    local hour=$(date +%H)
    if [ "${END_HOUR}" -gt "${START_HOUR}" ]; then
        [ "${hour}" -ge "${START_HOUR}" ] || [ "${hour}" -lt "${END_HOUR}" ]
    else
        [ "${hour}" -ge "${START_HOUR}" ] && [ "${hour}" -lt "${END_HOUR}" ]
    fi
}

run_backup() {
    local path="$1"
    local dest="${REMOTE_NAME}:backup/${USER}$(dirname ${path})"
    
    log "备份: ${path} -> ${dest}"
    
    # rclone命令说明:
    # copy: 复制，不删除源（我们用--remove-source-files删除）
    # --transfers: 并行文件数
    # --checkers: 并行校验数
    # --bwlimit: 带宽限制
    # --remove-source-files: 传输完成后删除源文件
    # --delete-during: 传输过程中删除已同步文件
    # --skip-symlinks: 跳过符号链接
    # --stats: 输出统计
    # --stats-one-line: 单行统计
    
    rclone copy "${path}/" "${dest}/" \
        --verbose \
        --transfers "${TRANSFERS}" \
        --checkers "${CHECKERS}" \
        --bwlimit "${BW_LIMIT}" \
        --checksum \
        --remove-source-files \
        --skip-symlinks \
        --log-file "${LOG_FILE}" \
        --stats 1s \
        --stats-one-line
    
    local exit_code=$?
    
    if [ ${exit_code} -eq 0 ]; then
        log "完成: ${path}"
    else
        log "错误: ${path} (退出码 ${exit_code})"
    fi
    
    return ${exit_code}
}

main() {
    mkdir -p "${LOG_DIR}"
    log "========== rclone备份开始 =========="
    
    if ! check_time_window && [ "${FORCE_BACKUP:-false}" != "true" ]; then
        log "跳过: 不在时间窗口内"
        exit 0
    fi
    
    # 测试连接
    log "测试连接..."
    rclone listremotes | grep -q "${REMOTE_NAME}"
    if [ $? -ne 0 ]; then
        log "错误: rclone remote '${REMOTE_NAME}' 未配置"
        exit 1
    fi
    
    # 执行备份
    for path in "${SOURCE_PATHS[@]}"; do
        if [ -d "${path}" ]; then
            run_backup "${path}" || log "备份失败: ${path}"
        fi
    done
    
    log "========== 备份完成 =========="
    
    # 输出统计
    rclone stats \
        --output-file "${LOG_DIR}/stats_$(date +%Y%m%d).txt" \
        --pickle-file "${LOG_DIR}/stats.pickle"
}

main "$@"

Systemd Timer配置​

# ~/.config/systemd/user/backup-rclone.service
[Unit]
Description=训练数据备份 (rclone)
After=network-online.target
Wants=network-online.target

[Service]
Type=oneshot
ExecStart=%h/scripts/backup-rclone.sh
Environment="FORCE_BACKUP=false"
StandardOutput=append:%h/.local/share/backup/logs/rclone.log
StandardError=append:%h/.local/share/backup/logs/rclone.error.log

[Install]
WantedBy=default.target
# ~/.config/systemd/user/backup-rclone.timer
[Unit]
Description=每天22:00自动备份训练数据

[Timer]
OnCalendar=*-*-* 22:00:00
Persistent=true
RandomizedDelaySec=15min

[Install]
WantedBy=timers.target

更高速方案：bbcp​

如果极致速度是首要需求，推荐 bbcp：

bbcp 优势​

特性bbcprsyncrclone
单文件速度9+ GB/s3-4 GB/s5-7 GB/s
多文件并行✅❌✅
内存占用低中中
断点续传✅✅✅
加密可选必须SSH可选

bbcp 安装​

# Ubuntu/Debian
sudo apt install bbcp

# 或源码编译
git clone https://github.com/ltp/bbcp.git
cd bbcp
make
sudo make install

bbcp 使用​

# 基本用法
bbcp -s -z -T "ssh -c aes128-ctr -i ~/.ssh/id_ed25519" \
    /data/training/dataset.tar.gz \
    user@nas:/backup/dataset.tar.gz

# 选项说明:
# -s: 静默模式
# -z: 压缩传输
# -T: 指定SSH命令（加密方式）
# -p: 保留权限
# -a: 显示进度

# 多文件并行传输
bbcp -s -z -T "ssh -c aes128-ctr" \
    /data/training/*.tar.gz \
    user@nas:/backup/

# 带校验
bbcp -s -z -V -T "ssh -c aes128-ctr" \
    /data/training/ \
    user@nas:/backup/

bbcp 完整脚本​

#!/bin/bash
# ~/scripts/backup-bbcp.sh
# bbcp高速备份脚本

set -euo pipefail

NAS_HOST="backup.nas.stars-labs.com"
NAS_USER="${USER}"
NAS_PATH="/backup/${USER}"
SOURCE_PATHS=(
    "/data/training"
    "/data/models"
)
BW_LIMIT=""  # bbcp自动适应
LOG_DIR="${HOME}/.local/share/backup/logs"

log() {
    echo "[$(date '+%Y-%m-%d %H:%M:%S')] $*" | tee -a "${LOG_DIR}/bbcp_$(date +%Y%m%d).log"
}

run_backup() {
    local src="$1"
    local dst="${NAS_USER}@${NAS_HOST}:${NAS_PATH}$(dirname ${src})"
    
    log "备份: ${src}"
    
    # bbcp命令:
    # -a: 保留属性
    # -z: 压缩
    # -v: 详细
    # -f: 强制覆盖
    # -w: 窗口大小 (16MB适合高RTT网络)
    # -T: SSH命令
    
    bbcp -a -z -f -w 64M \
        -T "ssh -c aes128-ctr -i ~/.ssh/id_ed25519 -o Compression=no" \
        "${src}/" \
        "${dst}/" \
        2>&1 | tee -a "${LOG_DIR}/bbcp_$(date +%Y%m%d).log"
    
    # 传输完成后删除源文件
    if [ $? -eq 0 ]; then
        log "删除源文件: ${src}"
        rm -rf "${src}"
    fi
}

main() {
    mkdir -p "${LOG_DIR}"
    log "========== bbcp备份开始 =========="
    
    for path in "${SOURCE_PATHS[@]}"; do
        if [ -d "${path}" ]; then
            run_backup "${path}"
        fi
    done
    
    log "========== bbcp备份完成 =========="
}

main "$@"

速度优化参数对比​

不同网络环境的最佳选择​

网络环境推荐技术关键参数
10Gbps局域网bbcp/rclone--transfers 16, -w 64M
1Gbps局域网rclone--transfers 8, --bwlimit 100M
100Mbps办公网rclone--transfers 4, --bwlimit 10M
跨公网rclone + 压缩--compress zstd, --bwlimit 5M
不稳定网络rclone--checkers 8, --retries 3

参数调优建议​

# 10Gbps局域网 - 极致速度
rclone copy src remote:dest \
    --transfers 16 \
    --checkers 32 \
    --bwlimit 0 \          # 不限制
    --buffer-size 256M \
    --fast-list

# 1Gbps网络 - 平衡速度
rclone copy src remote:dest \
    --transfers 8 \
    --checkers 16 \
    --bwlimit 100M \
    --buffer-size 128M

# 有丢包网络 - 可靠性优先
rclone copy src remote:dest \
    --transfers 4 \
    --checkers 8 \
    --bwlimit 50M \
    --retries 3 \
    --retries-sleep 1s \
    --no-check-certificate false

边传边删策略​

方案对比​

方式安全性速度影响推荐场景
--remove-source-files中无首次完整备份
--delete-during高轻微增量同步
自定义脚本高自定义特殊需求

推荐策略​

# 首次备份（完整复制，不删）
rclone copy src remote:dest \
    --transfers 8

# 后续备份（增量同步，边传边删）
rclone sync src remote:dest \
    --transfers 8 \
    --delete-during \
    --verbose

# 传输完立即删除源文件
rclone copy src remote:dest \
    --transfers 8 \
    --remove-source-files \
    --verbose

方案选择建议​

场景推荐方案原因
通用企业备份rclone功能全、易维护、多平台
极致速度要求bbcp单文件10GB/s+
跨公网传输rclone + 压缩断点续传、压缩
混合云存储rclone100+云后端支持
HPC环境bbcp高性能计算原生支持
