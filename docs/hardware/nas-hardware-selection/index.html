NAS 硬件选型采购方案

系统定位​

本方案针对 Linux + MinIO 对象存储 需求设计，支持分布式对象存储部署。

MinIO 硬件需求特点​

  • 多核CPU：MinIO 充分利用多线程
  • 大内存：缓存机制需要足够内存
  • 高带宽网络：2.5GbE/10GbE 推荐
  • 独立硬盘：每个硬盘独立挂载，无需RAID
  • SSD缓存：提升读写性能

设计目标​

  • 支持 ZFS 文件系统：适配 TrueNAS、飞牛OS 等 NAS 系统
  • 5盘位以上：满足 RAID-Z1/Z2 冗余需求
  • 热插拔支持：便于硬盘维护更换
  • 低功耗运行：7×24 小时运行，控制电费成本
  • 扩展能力：预留万兆网络、HBA 扩展空间

CMR vs SMR

ZFS 必须使用 CMR（传统磁记录）硬盘，避免 SMR（叠瓦式）硬盘。购买时确认型号是否为 CMR。

系统配置​

# MinIO 推荐系统配置
# 1. 禁用 swap
swapoff -a

# 2. 设置文件描述符
echo "* soft nofile 65536" >> /etc/security/limits.conf
echo "* hard nofile 65536" >> /etc/security/limits.conf

# 3. 优化网络参数
echo "net.core.rmem_max = 134217728" >> /etc/sysctl.conf
echo "net.core.wmem_max = 134217728" >> /etc/sysctl.conf
echo "net.ipv4.tcp_rmem = 4096 87380 134217728" >> /etc/sysctl.conf
echo "net.ipv4.tcp_wmem = 4096 65536 134217728" >> /etc/sysctl.conf

验收测试​

硬件检测​

  • CPU 温度正常（小于70°C）
  • 内存识别完整
  • 所有 SATA 接口识别
  • 网口 Link 状态正常
  • 热插拔功能测试

MinIO 测试​

  • MinIO 集群部署成功
  • 所有节点健康状态
  • 读写性能测试通过
  • 故障转移测试正常
  • 数据一致性验证

性能基准​

  • 单节点吞吐量大于 200MB/s
  • 集群总带宽接近网口带宽
  • SMART 状态健康
  • 7×24小时压力测试通过
