IT 服务支持与 SLA 标准

为保障公司业务高效运转，IT 部门建立以下服务支持流程与服务等级协议 (SLA)。所有员工在遇到 IT 问题时，请遵循本指南。

1. 服务渠道 (Service Channels)​

渠道适用场景响应时间联系方式
工单系统日常故障、权限申请、资产申领< 4小时Helpdesk Portal
IM 群组简单咨询、非紧急问题< 2小时Mattermost #it-support
紧急电话核心业务中断、网络瘫痪立即138-xxxx-xxxx (IT 值班)
邮件外部审计、合规通知< 24小时it-support@company.com

提示：为了便于追踪和审计，所有电话或 IM 沟通的故障，事后均需在工单系统补录。

2. 故障分级与 SLA (Service Level Agreement)​

我们根据故障对业务的影响程度，定义了四个优先级。

优先级定义 (Definition)示例 (Examples)响应目标 (Response)解决目标 (Resolution)
P0 - 紧急公司级业务中断，全员无法工作核心网断网、ERP 宕机、代码库不可用15 分钟4 小时
P1 - 高部门级业务受阻，影响多人会议室视频瘫痪、财务无法报销、某区域 WiFi 故障30 分钟8 小时
P2 - 中单人业务受阻，无临时替代方案个人电脑蓝屏、VPN 连不上、IDE 报错2 小时24 小时
P3 - 低体验问题或咨询，不影响核心工作显示器颜色偏、鼠标不灵、软件安装咨询4 小时3 工作日

3. 标准服务流程 (SOP)​

3.1 故障处理流程​

  1. 提交: 用户通过工单系统提交问题，包含截图和复现步骤。
  2. 分派: Helpdesk 一线客服在 15分钟 内进行初步研判和分派。
  3. 处理: 工程师接单处理，如需远程通过 Fleet/TeamViewer 需获得用户授权。
  4. 反馈: 处理完成后，工单系统自动发送邮件通知用户验证。
  5. 评价: 用户确认解决并对服务进行打分 (1-5星)。

3.2 权限申请流程​

  1. 提交: 用户填写《权限申请表》，注明系统 (GitLab/AWS/ERP) 和 角色。
  2. 审批: 直属主管审批 -> 数据Owner审批。
  3. 执行: IT 管理员在 Casdoor/LDAP 中添加组权限。
  4. 通知: 系统自动通知用户权限已生效。

4. 升级机制 (Escalation)​

如果故障在规定时间内未解决，将自动触发升级机制：

  • P0 级超时 1小时: 自动通知 CTO 和 IT 总监。
  • P1 级超时 4小时: 自动通知 IT 经理。
  • P2 级超时 24小时: 标记为 "Overdue"，计入工程师 KPI。

5. 维护窗口 (Maintenance Window)​

为了减少对业务的影响，常规的系统维护（如服务器重启、网络割接）将安排在固定窗口进行：

  • 常规维护窗口: 每周四 20:00 - 24:00
  • 紧急维护: 需 CTO 批准，并提前 30 分钟全员广播。

承诺: IT 部门致力于提供专业、高效的技术支持。您的每一次反馈都是我们改进的动力。
