# 投稿与贡献指南

欢迎参与 **IT 基础设施设计** 文档的共建！为了保持文档的条理清晰，请在投稿前参考以下目录结构规范。

## 📂 目录结构说明

我们将文档分为三个主要层级，请根据您要贡献的内容类型，将文件放入相应的文件夹：

### 1. 核心设计 (`01-core-design/`)
*   **内容类型**：顶层设计理念、整体架构图、全局软件栈选型、核心原则。
*   **适合投稿**：新的整体解决方案、企业级 IT 治理策略、开源软件栈的重大更新。

### 2. 硬件选型 (`02-hardware/`)
*   **内容类型**：具体的物理设备选型、参数对比、采购建议。
*   **适合投稿**：
    *   新的服务器型号评测
    *   网络设备（交换机/路由器）推荐
    *   存储硬件（硬盘/NAS）对比
    *   机房基础设施（UPS/空调/机柜）

### 3. 架构方案 (`03-architecture/`)
*   **内容类型**：具体的技术实施细节、网络拓扑、配置策略、运维流程。
*   **适合投稿**：
    *   网络规划（VLAN/路由/VPN）
    *   安全策略（防火墙规则/零信任）
    *   存储与备份策略
    *   高可用与容灾方案

## 📝 投稿步骤

1.  **Fork 本仓库**。
2.  在上述对应目录中新建 `.md` 文件。
3.  文件头部请添加 metadata (Front Matter)：
    ```yaml
    ---
    title: 您的文档标题
    sidebar_position: 5  # 可选，用于调整排序
    ---
    ```
4.  提交 **Pull Request**，并简要描述您的更改。

## 🤝 协作建议

*   **图片资源**：请将图片放入 `static/img` 目录，并在 Markdown 中使用绝对路径引用（如 `/img/your-image.png`）。
*   **图表绘制**：推荐使用 Mermaid.js 语法直接在 Markdown 中绘制流程图和架构图。

感谢您的贡献！
