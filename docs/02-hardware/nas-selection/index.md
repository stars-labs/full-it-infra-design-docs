# NAS 硬件选型概览

## 系统定位

本方案针对 **Linux + MinIO 对象存储** 需求设计，支持分布式对象存储部署。

### MinIO 硬件需求特点

- **多核CPU**：MinIO 充分利用多线程
- **大内存**：缓存机制需要足够内存
- **高带宽网络**：2.5GbE/10GbE 推荐
- **独立硬盘**：每个硬盘独立挂载，无需RAID
- **SSD缓存**：提升读写性能

## 设计目标

- **支持 ZFS 文件系统**：适配 TrueNAS、飞牛OS 等 NAS 系统
- **5盘位以上**：满足 RAID-Z1/Z2 冗余需求
- **热插拔支持**：便于硬盘维护更换
- **低功耗运行**：7×24 小时运行，控制电费成本
- **扩展能力**：预留万兆网络、HBA 扩展空间

## 推荐配置方案摘要

以下是针对不同规模的推荐配置，详细选型请参考后续章节：

### 方案 A：MinIO 入门版（6盘位 ~¥2000）
*   **适用场景：** 4-6节点 MinIO 集群，中小型企业对象存储
*   **核心配置：** 畅网 N305 主板 + 16GB DDR5 + 6盘位机箱

### 方案 B：MinIO 标准版（8盘位 ~¥2800）
*   **适用场景：** 8-12节点 MinIO 集群，高并发对象存储
*   **核心配置：** 畅网 N355 万兆主板 + 32GB DDR5 + 乔思伯 N4 机箱

### 方案 C：MinIO 企业级（8-10盘位 ~¥3500）
*   **适用场景：** 16+节点 MinIO 集群，生产级高可用
*   **核心配置：** 畅网 N355 万兆主板 + 32GB DDR5 + 10盘位机箱 + LSI HBA卡

### 方案 D：家用性价比（6盘位 ~¥1800）
*   **适用场景：** 个人家用，Plex/Emby 媒体中心
*   **核心配置：** 畅网 N305 主板 + 16GB 内存 + 恒煜机箱
