## 验收测试清单

### 硬件检测

- [ ] CPU压力测试(Prime95,满载运行4小时)
- [ ] 内存检测(memtest86通过)
- [ ] 磁盘健康检测(SMART状态正常,坏块检测)
- [ ] GPU压力测试(FurMark/CUDA-Z,满载运行2小时)
- [ ] 电源冗余测试(单电源故障切换)
- [ ] 散热测试(满载温度监控,GPU&lt;85°C)
- [ ] 噪音测试(满载噪音&lt;75dB)

### 软件环境检测

- [ ] 操作系统安装(Ubuntu 22.04 LTS / CentOS 8)
- [ ] GPU驱动安装(NVIDIA Driver 535+)
- [ ] CUDA环境(CUDA 12.1 / cuDNN 8.9)
- [ ] Docker环境(Docker 24.x + Docker Compose)
- [ ] 推理框架部署(vLLM测试模型加载)
- [ ] 向量数据库部署(Qdrant健康检查)
- [ ] 监控部署(Prometheus + Grafana)

### 性能基准测试

- [ ] 单GPU推理吞吐(DeepSeek-7B,tokens/sec&gt;50)
- [ ] 多GPU扩展效率(4× GPU &gt; 3× 单GPU性能)
- [ ] RAG检索延迟(P99 &lt; 500ms)
- [ ] 并发测试(50并发,成功率&gt;95%)
- [ ] 功耗监控(满载功耗符合标称)

## 采购流程建议

### 1. 需求确认
- 确定团队规模和并发用户数
- 明确计划运行的模型类型和参数量
- 确认预算范围和采购时间线
- 确定是否有国产化/信创要求

### 2. 供应商选择
- 评估2-3家供应商报价
- 查看供应商过往项目案例
- 确认售后服务响应时间
- 索取详细的技术方案

### 3. 技术评审
- 邀请技术团队参与评标
- 对比各方案的性能和扩展性
- 验证供应商的技术承诺
- 确认保修和服务条款

### 4. 合同签订
- 明确交付时间和验收标准
- 约定保修期限和服务级别
- 确定付款节点和质保期
- 包含技术培训要求

### 5. 预交付测试
- 要求供应商进行厂内测试
- 现场验收所有硬件组件
- 搭建基础软件环境
- 进行性能基准测试
